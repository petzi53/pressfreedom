## Project Overview

`pressfreedom` is an R package that bundles a cleaned, multi-year Reporters Without Borders (RWB) Press Freedom Index dataset and a Shiny dashboard for exploring it. The dashboard lets users compare country scores and rankings over time via interactive line charts and bump charts.

The companion project `rwb-book` (at `~/Documents/Meine-Repos/rwb-book/`) is a Quarto book documenting the full data pipeline. The cleaned dataset produced there is the source for the bundled `rwb` data object in this package.

Launch the dashboard with:

```r
pressfreedom::run_app()
```

## Bundled Dataset (`rwb`)

| Property | Value |
| :--- | :--- |
| Rows | 4,020 |
| Columns | 20 |
| Years | 2002вЂ“2025 (2011 missing) |
| Key columns | `year_n`, `score`, `rank`, `country_en`, `iso`, `political_context`, `economic_context`, `legal_context`, `social_context`, `safety` |
| Geography | Merged with UN M49 classifications (region, sub-region, country) |

Dimension scores (`political_context` etc.) are only available for 2022+.

### Known Data Issues

**2022 Zone Classification Anomaly** вЂ” In 2022, RWB used two non-standard geographic zones:
- "Europe - Asie centrale" (53 countries)
- "Maghreb - Moyen-Orient" (19 countries)

These zones appear **only in 2022** and do not align with the historical classifications used in all other years:
- "Europe - Asie centrale" countries belong to either "UE Balkans" (40) or "EEAC" (13) in other years
- "Maghreb - Moyen-Orient" countries belong to "MENA" (19) in all other years

**Resolution**: Applied in `data-raw/rwb.R` вЂ” 2022 zone assignments are automatically corrected during package build to match historical classifications. This preserves all 180 countries in 2022 while ensuring zone homogeneity across the entire time series.

**Factor Coercion Bug** вЂ” The upstream `rwb-book` pipeline sometimes encodes `country_en`, `zone`, and `iso` as factors. This previously caused subtle bugs downstream (e.g. mismatched `case_when()`/`%in%` comparisons against character values from Shiny inputs, and a factor value flowing unconverted into `plotly::plot_geo(locations = ...)`).

**Resolution**: Applied in `data-raw/rwb.R` вЂ” `country_en`, `zone`, and `iso` are explicitly coerced with `as.character()` right after loading the raw data, before any other transformation. All three columns are documented as `character` (not `factor`) in `R/data.R`. Downstream Shiny modules (`mod_inputs.R`, `mod_country.R`, `mod_chart.R`, `mod_map.R`) rely on this guarantee and no longer defensively re-coerce these columns themselves вЂ” if you add new code that reads `rwb$country_en`, `rwb$zone`, or `rwb$iso`, you should not need `as.character()` around it.

## Shiny App Architecture

The app uses a fully modular design (`inst/app/`):

- **`mod_inputs`** вЂ” sidebar with two `selectInput` widgets: variable type (score/rank) and country (multiple selection).
- **`mod_chart`** вЂ” renders a `plotly` card. Score в†’ scatter line chart; Rank в†’ `ggbump` bump chart converted via `ggplotly()`.
- **`helpers.R`** вЂ” `df_chart()` filters/prepares data; `card_title()` builds the dynamic card header.

`app.R` wires the modules and loads data via `pressfreedom::rwb`. All module files use explicit package namespacing (e.g., `shiny::`, `dplyr::`) so no additional `library()` calls are needed inside modules, except `library(ggplot2)` in `app.R` (required because `ggplotly()` resolves variables by name on the search path).

## Annual Update Workflow

Each May, RWB publishes a new index. To update the bundled dataset:

1. Re-run the data pipeline in `rwb-book`.
2. Re-run `data-raw/rwb.R` in this package.
3. Increment the version in `DESCRIPTION`.
4. Run `devtools::document()` and `devtools::check()`.
5. Run `renv::snapshot(type = "all")`.
